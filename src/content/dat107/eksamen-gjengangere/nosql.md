# NoSQL-Gjengangere

## Vanlige oppgavetyper

- lag et JSON- eller XML-dokument for et gitt domene
- bestem hvilke dokumenttyper eller collections som trengs
- vis hvordan et dokument lagres i PostgreSQL eller dokumentdatabase
- sammenlign dokumentdatabase med relasjonsdatabase
- begrunn valg av datamodell

## Hva som går igjen faglig

- semistrukturerte data
- innebygde dokumenter versus normalisering
- PostgreSQL som hybrid løsning
- fordeler og ulemper ved dokumenttenkning

## Typiske eksamensforventninger

- du må ikke bare lage et dokument, men også forklare hvorfor du har valgt akkurat den strukturen
- du må vise at du forstår hva som blir duplisert når data ikke normaliseres
- du må kunne si noe om når dokumentmodellen er praktisk og når den er problematisk

## Hva du bør kunne uten hjelpemidler

- forskjellen på XML og JSON på et overordnet nivå
- hvordan JSON eller XML kan lagres i PostgreSQL med én ID-kolonne og én datakolonne
- hvorfor innebygde dokumenter ofte er naturlige for ordre med ordrelinjer eller lignende hierarkier

## Hva du bør øve på

- tegne én relasjonell løsning og én dokumentløsning av samme case
- skrive kort begrunnelse for valg av dokumentstruktur
- lage ett konkret eksempel på JSON eller XML med realistiske data

## Siste eksamenstips

En god NoSQL-besvarelse er nesten alltid balansert: du må vise både hvorfor dokumentmodellen passer og hva du mister sammenlignet med relasjonsmodellen.
