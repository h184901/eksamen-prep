# Oblig 1: SQL

## Hva denne obligen trener

- tabellutforming fra tekstbeskrivelse
- PK, FK, datatyper og `null`
- `alter table`
- `join` og `left join`
- relasjonsalgebra
- telling og siste registrerte hendelse

## Dette må du kunne

- forklare hvordan passering uten lest skilt bør lagres
- skrive spørringer som både tar med og utelater slike tilfeller
- begrunne tabellvalg, ikke bare skrive kode

## Vanlige feil

- ikke modellere spesialtilfellet med manglende registreringsnummer tydelig nok
- velge `join` når oppgaven egentlig krever `left join`
- glemme å lage testdata som dekker spesialtilfellet

## Typiske eksamensoppgaver

- SQL-spørringer med `null`
- telling per gruppe
- kombinasjon av DDL, DML og spørring i samme case

## Hva du bør øve på

- lage egne testdata med minst én passering uten skilt
- skrive både SQL og relasjonsalgebra for samme idé
