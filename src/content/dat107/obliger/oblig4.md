# Oblig 4: NoSQL

## Hva denne obligen trener

- XML i PostgreSQL som hybrid løsning
- migrering fra relasjonelle tabeller til XML og JSON
- dokumenttenkning i MongoDB
- enkel CRUD i Java over MongoDB

## Dette må du kunne

- forklare hvorfor `KUNDE_NY` og `ORDRE_NY` blir dokumentorienterte varianter av relasjonelle tabeller
- forklare hvorfor `ordre` passer som dokument med innebygde ordrelinjer
- se forskjellen på teknisk migrering og datamodellvalg

## Vanlige feil

- fokusere bare på syntaks i stedet for å forklare hvorfor strukturen er valgt
- glemme at dokumentmodellen gir duplisering og andre trade-offs
- ikke koble XML/JSON-løsningene tilbake til relasjonelt utgangspunkt

## Typiske eksamensoppgaver

- lag dokumentdatabase for et domene
- vis eksempel på JSON eller XML-dokument
- drøft dokumentdatabase versus relasjonsdatabase

## Hva du bør øve på

- beskrive samme case både relasjonelt og dokumentorientert
- begrunne hvilken løsning du ville valgt på eksamen og hvorfor
