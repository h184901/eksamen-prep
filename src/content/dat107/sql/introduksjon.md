# Introduksjon

## Kjernen

SQL er språket vi bruker for å:

- definere tabeller og begrensninger
- legge inn, endre og slette data
- hente ut og kombinere data
- styre transaksjoner

I DAT107 er poenget ikke bare å kunne skrive syntaks, men å kunne gå fra tekstbeskrivelse til riktig databasevalg og riktig spørring.

## Dette må du kunne

- forklare hva en tabell, rad, kolonne og nøkkel er
- skille mellom DDL, DML og spørringer
- forstå hvordan SQL henger sammen med datamodellering og normalisering
- lese oppgavetekst og se hvilke tabeller og koblinger som trengs

## Nøkkelbegreper

- database
- DBMS
- skjema
- tabell
- primærnøkkel
- fremmednøkkel
- relasjon
- transaksjon

## Vanlige feil

- tenke på SQL som bare kode, ikke som modell + kode
- skrive spørringer uten å forstå relasjonene mellom tabellene
- glemme at riktig resultat er viktigere enn bare gyldig syntaks

## Typiske eksamensoppgaver

- forklare sentrale begreper
- gå fra tekst til tabeller og nøkler
- velge passende datatyper og begrunne dem

## Hva du bør øve på

- forklare med egne ord hva primærnøkkel og fremmednøkkel gjør
- skille mellom strukturkommandoer og spørringer
- bruke en liten tekstbeskrivelse og skissere aktuelle tabeller før du skriver SQL
